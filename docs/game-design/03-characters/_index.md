## 호감도 시스템

### 호감도 범위

| 수치 | 단계 | 설명 |
|------|------|------|
| -100 ~ -50 | 적대 | 대화 거부, 방해 |
| -49 ~ -1 | 경계 | 차가운 반응 |
| 0 ~ 19 | 중립 | 기본 상태 |
| 20 ~ 39 | 호의 | 친절한 반응 |
| 40 ~ 59 | 친밀 | 개인적 대화 가능 |
| 60 ~ 79 | 신뢰 | 비밀 공유 |
| 80 ~ 100 | 깊은 유대 | 특별 이벤트 |

### 호감도 영향 요소

| 행동 | 효과 |
|------|------|
| 도움 | +5~15 |
| 대화 | +1~5 |
| 선물 | +3~10 |
| 비밀 공유 | +10~20 |
| 무시 | -3~10 |
| 거짓말 | -10~20 |
| 배신 | -30~50 |

### 루프 간 유지

- 호감도는 **영구 데이터**로 루프 간 유지됨
- NPC는 기억 못 하지만, 플레이어의 "인상"은 남아있음
- 게임적 표현: "왠지 모르게 친근하다" / "왠지 불편하다"

## 캐릭터 파일 템플릿

각 캐릭터 파일은 다음 구조를 따름:

```markdown
# [캐릭터 이름]

## 기본 정보
- ID: `character_id`
- 이름:
- 나이:
- 직급:
- 소속:

## 외모
간략한 외모 설명

## 성격
성격 특성 및 특이점

## 배경 스토리
과거사 및 현재 상황

## 게임 내 역할
스토리/게임플레이에서의 기능

## 호감도 이벤트
호감도 단계별 해금 이벤트

## 대사 예시
캐릭터 말투 예시

## 관련 단서
이 캐릭터를 통해 얻는 단서
```

## 캐릭터 디자인 원칙

### 1. 공감 가능성
- 모든 캐릭터는 현실에서 만날 법한 사람
- 과장된 설정보다 현실적 고민과 욕망

### 2. 다층적 깊이
- 표면: 첫인상, 직장에서의 모습
- 이면: 숨겨진 과거, 진짜 감정
- 비밀: 스토리 반전과 연결

### 3. 기능적 역할
- 각 캐릭터는 게임플레이 기능 수행
- 정보 제공, 관계 발전, 스토리 진행

### 4. 상호 연결
- 캐릭터 간 관계가 얽혀있음
- 한 캐릭터의 정보 → 다른 캐릭터 이해에 도움
